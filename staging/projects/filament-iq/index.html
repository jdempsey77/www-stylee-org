What it is

Filament IQ connects a Bambu Lab P1S printer — Automated Material System(AMS) — to Home Assistant and Spoolman. After every print it tracks filament consumption: using the RFID hardware delta for RFID-tagged spools, and 3MF slicer estimates for non-RFID spools.

Every print event flows through a deterministic five-phase pipeline: Collect → Decide → Execute → Notify → Finalize. The decision engine is pure Python with zero I/O — it accepts typed input and returns a decision.

Key Features

  • RFID spool auto-identification — reads tray UUID directly from AMS sensors; RFID delta always wins over slicer estimates
  • Five-phase pipeline — Collect → Decide → Execute → Notify → Finalize; pure Python decision engine with full test coverage
  • Filament IQ Manager — custom Lovelace card (Preact + Vite, ~51 KB); spool inventory with color dots, progress bars, material badges, location badges; inline CRUD; SpoolmanDB fuzzy search across 6,957 filaments
  • Proxy architecture — filament_iq_proxy HA custom component routes Spoolman REST API calls through HA’s authenticated WebSocket; no unauthenticated ports exposed

AppDaemon is synchronous — Filament IQ uses urllib.request-based HTTP calls, not asyncio. The decision engine (consumption_engine.py) is a pure function: zero AppDaemon dependency, accepts SlotInput dataclasses, returns SlotDecision dataclasses, no I/O.
